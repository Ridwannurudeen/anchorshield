# M6 Mainnet & SDK

## Scope Completed

M6 adds the developer-facing surface that can be completed without external approval:

- generated TypeScript bindings for `gate_payment` and `gate_rwa`
- `packages/sdk` public-signal/action-binding helpers
- `packages/cli` local `anchorshield` CLI
- `docs/SDK.md`
- `docs/MAINNET_RUNBOOK.md`
- `scripts/m6-verify.ps1`

## Verified Binding Workflow

Local CLI:

```bash
stellar 27.0.0
```

Verified command shape:

```bash
stellar contract bindings typescript --wasm <WASM> --output-dir <OUTPUT_DIR> --overwrite
```

Generated outputs:

- `packages/bindings/gate-payment`
- `packages/bindings/gate-rwa`

The generated bindings use `@stellar/stellar-sdk@^14.5.0`; its contract type definition maps `u256` to `bigint`.

## SDK

`packages/sdk` covers:

- parsing the 17-field public signal vector
- formatting converter `[{ "u256": "..." }]` public signals
- formatting implicit Stellar CLI plain-array public signals
- formatting generated-binding `bigint[]` public signals
- converting Groth16 proof/VK hex strings into binding-ready buffers
- asserting payment and RWA action-binding
- building `verify_and_pay` and `verify_and_transfer` argument objects
- local proof generation/verification wrappers around the pinned `snarkjs@0.7.6`

## CLI

The local CLI supports:

- `inspect-public`
- `validate-action`
- `soroban-args`
- `events`
- `disclosure verify`
- `gate payment`
- `gate rwa`

The `gate` commands print a Stellar CLI invocation. They do not submit transactions.

The printed invoke command uses the current implicit CLI's verified argument-file flags:

- `--vk-file-path`
- `--proof-file-path`
- `--pub_signals-file-path`

`stellar contract invoke` also requires `--source-account`; the CLI accepts one or prints `<SOURCE_ACCOUNT>` as a placeholder.

## Verification

```bash
npm run m6:verify
```

This runs JS syntax checks, SDK tests, CLI tests, and package JSON checks.

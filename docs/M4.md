# M4 Compliance & Disclosure

## Scope

M4 adds:

- `services/disclosure/disclosure.js`: generates an auditor X25519 view key, encrypts a Travel-Rule packet with AES-256-GCM, decrypts it as the auditor, and verifies binding to the payment proof's public `packet_hash`.
- `services/indexer/build-index.js`: normalizes M1/M2 contract events fetched from Stellar testnet into a no-PII compliance event index.
- `apps/web/data/disclosure-summary.json`: auditor verification summary rendered in the public dashboard.
- `apps/web/data/compliance-events.json`: indexed compliance event trail rendered in the public dashboard.

## Commands

```bash
npm run m4:disclosure
npm run m4:index
```

Both commands passed.

## Disclosure

The disclosure flow:

1. Builds a Travel-Rule packet from `testdata/eligibility/input.valid.json`.
2. Binds it to public `packet_hash` from `testdata/eligibility/public.json`.
3. Encrypts the packet to an auditor X25519 public key.
4. Decrypts with the generated auditor private view key.
5. Recomputes the circuit public signals with `snarkjs.groth16.fullProve`.
6. Verifies the recomputed `packet_hash`, payment tx, action id, amount, and corridor.

## Indexer

Verified CLI interface:

```bash
stellar tx fetch events --hash <tx> --network testnet --output json
stellar events --network testnet --id <contract> --output json
```

M4 uses exact tx event fetches for deterministic replay:

- Payment event raw source: `services/indexer/raw/payment-events.json`
- RWA event raw source: `services/indexer/raw/rwa-events.json`
- Normalized output: `services/indexer/compliance-events.json`

The indexed events include policy id, action id, asset id, amount, recipient field, nullifier, packet/terms hash, action binding, tx hash, and Stellar Expert link. They do not include raw KYC, sanctions, originator, or beneficiary fields.

# M1 Core Primitive

## Scope

M1 implements the first executable AnchorShield gate:

- `circuits/eligibility.circom`: BLS12-381 Circom proof of credential membership, KYC/sanctions flags, country, age, investor type, transaction limit, expiry, packet binding, nullifier, and action binding.
- `contracts/gate_payment`: Soroban proof gate with co-located M1 issuer root, policy, nullifier, verifier, and mock-asset payment ledger.
- `testdata/eligibility`: non-secret proof, verification key, public signals, converted Soroban CLI args, and deterministic input fixture.

Mock scope is explicit: KYC and sanctions are issuer-attested fields, the asset is an internal mock ledger, and the issuer service is currently a deterministic fixture rather than an HTTP service.

## Circuit

The circuit uses:

- `Poseidon255` and `MerkleProof` copied from the verified Stellar `privacy-pools` example.
- `circomlib@2.0.5` comparators and bitify components.
- `--prime bls12381` with `circom 2.2.3` and `snarkjs 0.7.6`.

Public signal order:

1. `credential_root`
2. `packet_hash`
3. `nullifier`
4. `action_binding`
5. `issuer_id`
6. `policy_id`
7. `kyc_required`
8. `sanctions_required`
9. `allowed_country`
10. `min_age`
11. `min_investor_type`
12. `action_type`
13. `asset_id`
14. `amount`
15. `recipient`
16. `action_id`
17. `epoch`

## Contract

Contract path: `contracts/gate_payment`

The contract enforces:

- registered credential root equals public `credential_root`
- policy fields equal the public proof inputs
- real action args equal the public proof inputs
- packet hash arg equals public `packet_hash`
- nullifier is unused before payment and marked used after payment
- Groth16 pairing verification succeeds
- mock escrow has enough balance before crediting the mock recipient

Local tests:

```bash
cd contracts/gate_payment
cargo test -- --nocapture
```

Result: `6 passed`.

Covered paths:

- valid proof executes mock payment
- proof for action A fails for action B
- wrong policy parameters reject
- packet hash mismatch rejects
- reused nullifier rejects
- wrong registered root rejects

## M0 Gate

Do not build custom AnchorShield contracts or circuits until:

1. The upstream `groth16_verifier` builds locally. Done in WSL.
2. The upstream positive and negative proof tests pass locally. Done in WSL.
3. A verifier contract deploys to Stellar testnet. Done.
4. A proof verification is invoked against the testnet deployment. Done with `--send yes`.
5. AnchorShield has golden-vector tests for converting the upstream snarkjs JSON proof, verification key, and public signals into Soroban BLS12-381 structures. Done in `tools/groth16-json-converter`.

## Converter Notes

- `proof.json` and `verification_key.json` are converted from snarkjs decimal coordinates into Soroban CLI hex bytes using Arkworks BLS12-381 uncompressed serialization.
- The Stellar CLI help says `pub_signals` accepts `["1"]`, but file-mode invocation required explicit SCVal-style `[{ "u256": "33" }]`.
- `tools/groth16-json-converter` tests assert point byte lengths and the deterministic SHA-256 hash of the converted fixture JSON.

# M3 Real UX

## Scope

M3 adds a browser demo dApp:

- `apps/web`: static demo surface.
- `apps/web/proving/eligibility.wasm`: compiled circuit witness generator.
- `apps/web/proving/eligibility_final.zkey`: M2 smoke proving key.
- `apps/web/vendor/snarkjs.min.js`: installed `snarkjs@0.7.6` browser bundle.
- `apps/web/data`: payment/RWA witness inputs, verification key, and testnet deployment record.

The app generates Groth16 proofs in the browser for both M1 payment and M2 RWA flows, verifies them locally with `snarkjs`, checks public inputs against the real action arguments, and runs mismatch tests that mutate public inputs and reject proof reuse.

## Local Run

```bash
npm run m3:web
```

Local URL: `http://127.0.0.1:4173/`

## Public Demo

- Scoped nginx location added under the existing `preflight.gudman.xyz` HTTPS vhost.
- `nginx -t` passed and nginx was reloaded.

Notes:

- `anchorshield.gudman.xyz` did not resolve, so no new subdomain/certificate was created.
- The initial copy left the `proving/` directory as `700`; permissions were corrected to directories `755` and files `644`.

## Browser Verification

Playwright checks:

- Local desktop page loaded with no console errors after favicon patch.
- Local payment proof generated and verified in-browser in `3624ms`.
- Local RWA proof generated and verified in-browser in `3430ms`.
- Local mismatch checks rejected mutated payment amount and mutated RWA recipient public inputs.
- Local mobile layout at `390px` stacked cleanly with no overlap in the accessibility snapshot.
- Public hosted RWA proof generated and verified in-browser in `9859ms` over HTTPS.

## Wallet

The UI loads Freighter's browser API from the documented CDN package path and attempts `isConnected` / `requestAccess` when the wallet button is pressed. In browsers without Freighter, the app reports wallet unavailable and still supports local proof generation.

## Honest Scope

The browser currently proves and verifies locally, then links to the already executed M1/M2 testnet transactions on Stellar Expert. It does not yet build, sign, and submit Soroban transactions from the browser; that remains part of the later SDK/CLI/mainnet work.
